Quelques notes et regles a propos du code

1.1) Les fonctions/procedure doivent être dans un namespace ::PREEDb0t::, c'est a dire commence par ::PREEDb0t:: 

2.1) Protection des variables avec {}. par exemple $chan deviens ${chan}. une bonne habitude a prendre
2.2) Les variables qui doivent être disponible d'un procedure/fonction a une autre (a des endrois differents) 
        doivent être declarer dans le namespace ::PREEDb0t:: (dans preed.tcl avec variable ...)
2.3) Suite regle 2.2, les variables d'une proc a l'autre doivent commencer par ::PREEDb0t:: par exemple $mysql deviens ${::PREEDb0t::mysql}

3.1) si dans un if, il ya plusieur condition, toujours commencer par celle qui a le plus de chance d'arriver (en negation comme afirmation) 
3.2) si plusieurs if sont d'affiler, toujours mettre le if qui a le plus de chance darrivé en premier

4.1) utilisation de l'identation 'tab' en taille 4

5.1) il est de bonne pratique de proteger les tables et colonnes sql avec des ``, par exempe (select col from db) deviens  (select `col` from `db`)

6.1) Pour les messages IRC, si nous utilisons [format "aaa %s aaa" $var] plus tard ont poura faire des message personalisable par utilisateur
